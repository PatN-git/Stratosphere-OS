# UX Design

**Hand-off contract:** Writes a path-aware design document to `docs/design/BT-<padded>-ux.md` by instantiating `.agents/workflows/.reference/design-doc-template.md`. Appends design path to parent GitHub issue body. Downstream workflows (`3a`, `3c` Phase 0, `4a`) read this document.

---

## Phase 1: Surface & Scope Gate
1. Load PRD from `docs/prds/BT-<padded>-<feature>.md`.
2. Choose one execution path based on the feature's interface surface:
   - **Path C: Non-UI / Headless Feature.** If the feature has NO UI component/visual interface (e.g. backend api, workers, database-only changes, libraries). Skip visual design and go directly to Path C interface contract.
   - **Path B: UI Feature, No Stitch.** If the project does not use Google Stitch or you are designing manual layouts. Go to Path B manual blueprint.
   - **Path A: UI Feature, Stitch-Assisted.**
     - **Branch A1: Whole-page (New Page or Full Redesign).** Stitch authors the entire page. Defer to §3 Immortal Components for any global shell chrome, but ADOPT and register new layout blocks as §3 Immortal Components.
     - **Branch A2: Feature on Existing Page.** Stitch lays out the feature-body region only. Existing global chrome/components (Navbar, Sidebar, Footer) are SHIELDED and remain untouched (DR-008/010).
3. If the project has NO external surface/code changes (e.g. documentation-only, process change), skip this workflow entirely and handoff to `/3a_create-issue`.

## Phase 2: Design Doc Initialization
1. Every path must instantiate a design document at `docs/design/BT-<padded>-ux.md` from `.agents/workflows/.reference/design-doc-template.md`.
2. Populate the shared metadata shell:
   - `slug`: feature kebab-case slug
   - `bt`: `BT-<padded>`
   - `prd`: path to PRD
   - `surface`: `ui-stitch-page` | `ui-stitch-feature` | `ui-manual` | `non-ui`
   - `status`: `draft`
   - `updated`: current date
3. Keep exactly ONE body block in `docs/design/BT-<padded>-ux.md` (Path A, Path B, or Path C block) and delete the other two blocks.

## Phase 3: Harmonize & Freeze (Path Execution)

### Path A: Google Stitch
1. **Ingest Layout:** Ingest the layout generated by Stitch. Use the Stitch MCP server if connected; otherwise, copy/paste the exported layout markup/tokens directly into your context. (Graceful degradation: MCP is an accelerator, never a runtime dependency).
2. **Token Snap:** Extract all visual/layout tokens from the design and snap them to the CSS variables / Tailwind tokens defined in `.memory/DESIGN.md` (following DR-009).
3. **Immortal Components Reconciliation:**
   - For Whole-page branch (A1): Propose adding the page-level structure (chrome/shell) to `.memory/DESIGN_RULES.md` §3 Immortal Components (LAW-tier, to be committed via setup or 0b).
   - For Feature-on-existing branch (A2): Discard any modifications proposed by Stitch to existing global shell components (shielding per DR-008/DR-010).
4. **Freeze Layout Blueprint:** Write the resolved layout regions, DOM hierarchy, and component placements into `docs/design/BT-<padded>-ux.md` under the Path A body block.
5. Set `status` in frontmatter to `ready-for-slicing`.
6. **Read Rule:** Stitch is read ONCE at design time and frozen. Downstream workflows (`3a`, `3c`, `4a`, `4b`) read ONLY the frozen design doc, `.memory/DESIGN.md`, and `.memory/DESIGN_RULES.md`. Never access live Stitch during implementation or audit.

### Path B: Manual UI
1. Define the spatial layout rules, component hierarchy, visual tokens, and breakpoints.
2. Write the details into `docs/design/BT-<padded>-ux.md` under the Path B body block.
3. Set `status` in frontmatter to `ready-for-slicing`.

### Path C: Non-UI Interface Contract
1. Define the API signatures, schema definitions, I/O parameters, behavioral invariants, and input edge-case matrix.
2. Write the details into `docs/design/BT-<padded>-ux.md` under the Path C body block.
3. Set `status` in frontmatter to `ready-for-slicing`.

## Phase 4: Publish & Sync
1. Append the design reference link (`docs/design/BT-<padded>-ux.md`) to the parent GitHub Issue body.
2. Append the design document reference to the `Ref` column in `.memory/BACKLOG_MAP.md` alongside the PRD link.
3. Handoff to `/3a_create-issue`: *"UX Design completed. Run `/3a_create-issue` to slice requirements."*
